**** openssl genrsa -out keypair.pem 2048

Генерация RSA Ключевой Пары:
openssl genrsa: команда для генерации RSA ключевой пары.
-out keypair.pem: имя файла, в который будет сохранена сгенерированная ключевая пара.
2048: создается 2048-битный ключ, что считается достаточно безопасным для большинства целей.


 **** rsa -in keypair.pem -pubout -out public.pem
 writing RSA key

Экспорт Открытого Ключа:
openssl rsa: команда для обработки RSA ключей.
-in keypair.pem: файл, из которого будет читаться RSA ключевая пара.
-pubout: экспортировать открытый ключ из указанной ключевой пары.
-out public.pem: файл, в который будет сохранен открытый ключ.


**** pkcs8 -topk8 -inform PEM -outform PEM -nocrypt -in keypair.pem -out private.pem

Преобразование Закрытого Ключа в Формат PKCS#8:
openssl pkcs8: Команда для обработки закрытых ключей в формате PKCS#8.
-topk8: преобразовать входные данные в формат PKCS#8.
-inform PEM:формат входного файла (PEM — Privacy Enhanced Mail).
-outform PEM: формат выходного файла, который также будет PEM.
-nocrypt: файл не должен быть зашифрован. Если нужно зашифровать, то используются другие опции для указания метода шифрования и пароля.
-in keypair.pem: файл, из которого считывается закрытый ключ.
-out private.pem: файл, в который будет сохранен преобразованный закрытый ключ.

------------------------------------------------------------------------------------------------
PKCS#8 (Public-Key Cryptography Standards #8) — это стандарт,
который описывает общий формат для закрытых ключей, включая информацию о
шифровании ключа. Этот формат поддерживается многими программными и аппаратными
системами, обеспечивая более широкую совместимость по сравнению с другими форматами,
такими как PKCS#1, который специфичен только для ключей RSA.
